Open SSL 취약점(Heartbleed) 검사 실시

내용

최근 Open SSL 취약점(HeartBleed)에 대하여 보도되었습니다.
( 보안뉴스 참조 : http://www.boannews.com/media/view.asp?idx=40527&kind= )

이 취약점은 HeartBleed로 명명되어 있습니다.
이를 악용할 경우 웹서버로 전송된 개인정보, 비밀번호 등은 물론 웹서버의 암호키도 탈취횔 수 있는 것으로 알려져 있습니다.
각 사업부 및 개발팀에서는 다음과 같이 조치 부탁드립니다.

<조치절차>
1. 대상 : SSL이 적용되어 있는 모든 사이트
2. 검사방법 :
- https://filippo.io/Heartbleed 접속
- 서비스 URL 입력 후 Enter 입력 (예 : www.webcash.co.kr)
- 'All good, ~~ '으로 시작하는 메시지가 나오면 문제 없음
3. 결과보고 :
- 확인 결과를 '솔루션부문 > R&D 기획실' 김이슬에게 메일 전달
4. 문제 발생 시 '솔루션부문 > DB R&D실' 백청석 팀장에게 문의
